Detailed Description

構文解析イベントの種類を表す．

構造文書を表す木の深さ優先巡回（traversal）を行った際に、各ノードで次のようにイベントを生成する。

  • 根ノードを訪問したときに、行きがけ順と帰りがけ順のそれぞれで、SDOCとEDOCイベントが生成される。
  • 根以外の内部ノードを訪問したときに、行きがけ順と帰りがけ順のそれぞれで、STAGとETAGイベントが生成される。
  • 葉ノードを訪問したときにTEXTイベントが生成される。

一般に、半構造文書のストリーム処理においては，イベントの種類に基づいて，その頂点に関する何らかの前処理を行い，部分木の処理を終えて，再び帰りがけ順で同じ頂点にもどってきた際に，その頂点でクエリのフィルタ （述語）の評価を行い，その頂点（とその部分木）を解として出力す るかどうかを決める．

  • また，この評価は，ボトムアップに葉から根に向けて，再帰的に行われて，最終的な出力結果が決まる．

GUARD

訪問中の頂点の子供列中の「ガード位置」の検出を表すイベント型.


文書の行きがけ順（pre-order）の訪問を仮定している. XML文書の場合は，開始タグ中の属性要素列の終端で検出される．JSONなど、属性頂点を持たない半構造文書では検出されない． ガードは、その頂点の子供列を左から右に操作している際に，その部分木の枝刈りのための条件判定を行って良い位置を表す．もしGUARD位置の子供でクエリのフィルタ（述語）が失敗すれば，残りの子供の操作はせずに（つまり帰りがけ順の位置を待たずに）親頂点を失敗させて良い．パターン照合の性能向上のために利用しても良いが，無視しても照合の正当性には影響しないように利用することが望ましい．
