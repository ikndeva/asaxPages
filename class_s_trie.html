Detailed Description

接尾辞トライによるテキスト索引のクラス。

  • 本索引では、各辺は一つの文字を表す原子辺（atomic edge）である。
  • 入力テキストを保持せず、木に文字を格納するため、長さ$n$文字のテキストの全接尾辞を保持するのに$O(n^2)$メモリを要する。

索引構築の主手続き。

  • 文字列を受け取り、その全ての接尾辞を挿入した索引を構築する。
  • Ukkonennによる接尾辞トライ構築の$O(n^2)$ 時間アルゴリズム。
  • テキストを左から右に走査しながら、各文字 ch について次のプロセスを繰り返すことで、すべての接尾辞を挿入する。
  • 繰り返しでは、各文字 ch ごとに、最長接尾辞に対応する葉から開始して、接尾辞リンクを根までたどりながら、現在のノードからch-枝を拡張する。
