Detailed Description

論理位置アクセスによる再読み出しが可能なバッファつき入力ストリーム.

入力ストリームは、過去のある時点から現在までに読み込んだ入力ストリームの内容をバッファに保持している. さらに、バッファー上に、元の入力ストリーム上の現在の読み込み位置を表す{read}カーソルと、自由に移動可能な{rescan}カーソルの二つのカーソルをもつ. バッファー中のカーソルの位置は、範囲[0..MAXBUF]の整数である論理位置で指定される. （ここに、論理位置は、必ずしも元のストリーム上の先頭からの文字数と一致するとは限らないので注意されたい. ） 第一に、{read}カーソルに対しては、@javalink read}メソッドで読み込みが可能である. 第二に、{rescan}カーソルに対しては、@javalink resetCursor}メソッドで任意の論理位置に移動した上で、rescan, rescanForward, rescanBackwardなどのメソッドを用いて、{read}カーソルと独立に読み込みが可能である. 典型的な用途として、先頭と終了の論理位置の対を適宜記録することで、バッファに格納されている範囲で、入力文字列中の部分文字列の切り出しなどが実現できる.

◆ MAXBUF

int MAXBUF = 8192
static

読み込み済みの入力を保持するためのバッファの最大バッファ長.

この長さを超える部分文字列は取り出せないので、要素の最大テキスト長より大きく設定すること. 既定値は{BLKSIZE = 8192}文字である. {Match}コマンドでのoption -m でも設定可能.

バッファを1回にまとめて埋めるブロックサイズ.
