210507 Design of Syntactic Objects

目的

XScan プログラムの構文的な構成要素（syntactic elements）を設計する．

プログラムの実行により，メモリ上に意味的な対象である実体（entity）が生成・配置される．

実体は，構文的な要素である名前で指し示すことができる．名前をもたない実体もある．

プログラムの解析は次を行う：

  • 構文解析： 構文要素を取り出し，あとの使用のために組織化し，格納する．
  • 意味解析： 構文要素に生成されるべき実体の情報を対応させ，実態に関する解析を行う．

構成

プログラムは，各種のオブジェクトをもつ．

オブジェクトは，次の2種がある．

  • 静的オブジェクト： プログラムで構文的に決定するもの．構文オブジェクト．
  • 動的オブジェクト： 実行過程で手続きで生成し，動的にメモリ上に配置されるもの．実体オブジェクト．

A. 構文的要素

構文的な名前全ては，ユニバース（universe）と呼ばれる，大域的な辞書に格納され，連番管理される．

名前

名前は，静的または動的な対象（モノ）を指し示す． 名前が，**大域的**（global）とは，それがプログラム全体で一意であることをいう．ただし出現は複数あって良い．

  • 関数名： インターンした関数名文字列のIDである．補助情報をもてる．
  • 変数名： インターンした変数名文字列のIDである．補助情報として，型をもてる．
  • 定数名： 単一代入のみ可能な変数として扱う．

名前は，ある実体ですべて大域的であり，同じ名前文字列は，大域記号表にinternされて，識別子（identifier, ID）に変換されて用いられる．名前文字列そのものはプログラムでは用いられない．

名前の生成は，次の2箇所で行われる．

  • ソースコードに出現する文字列（名前文字列）からのインターン時
  • 無名の構文オブジェクトの生成時．系統的に，名前文字列が生成される．

リテラル

構文レベルで，値の記述法を提供する仕組み．

  • Integerリテラル: いわゆる整数表記．例：0, 128, -24.
  • Doubleリテラル: いわゆる実数表記．例：01L, 3.1415.
  • Booleanリテラル: いわゆる真偽値．例：true, false.
  • 文字リテラル：未定義．例：a, \n, \u0000, \u007F.
  • 文字列リテラル: 二重引用符，単一引用符，スラッシュ引用符．f,q,r演算子で記述された文字のならび．例："A Book", ‘'a pencile’,/(Mon|Tue|Wed|Thu|Fri|Sat|Sun)/,f/{ yy }{ mm }{ dd }/,q/2020.12.24/.
  • パスリテラル： ドット演算子，ドット閉包演算子で記述されたタグ名の並び．例：.dblp.article`

その他，将来的には次のリテラルの導入を検討する．

  • タプルリテラル： 多値返り値
  • 辞書リテラル： ディスパッチャー，構造体，モジュール
  • リストリテラル： 値の列

ブロック

ブロックは，コードの範囲を表す構文要素である．

プログラムは，「トップブロック」と呼ばれる，実行環境（計算機）全体を表す特別なブロックを一つもつ．

トップブロック以外のブロック全ては，必ず親となるブロックを一つもつ． したがって，全てのブロックは単一の根をもつ木をなす．

ブロックは，実体を指し示す一意なブロックポインタ（BlockRef）をもつ．

package univ; //ユニバース
interface Block {
Block(Block parent_block); //唯一のコンストラクタ
int getParent(); //トップブロック以外のブロック全ては，非nullの親ブロックをもつ．
int getVarTable(); //ブロック全ては，必ず非nullのVarTableを一つもつ．
}
interface VarTable {
VarTable(VarTable parent_table); //唯一のコンストラクタ
}
interface Universe {
Block createBlock(Block parent);
Var createVar(Block block);
}

B. 実体的要素

???

プロセスは次の実体をもつ

  • 関数： 関数の実体を指し示す構造体へのポインタである．コンパイル時に生成・決定される．
  • 変数： 変数の実体であるメモリ上のハコ（cell）の番地である．実行時に動的に生成・決定される．

それぞれの実体が指定されると，処理の実行や，値の格納とアクセスなどの必要な機能を提供できる．

値

値は，メモリ上にある時点で置かれた情報であり，究極ではメモリ上に組織されたビットパターンで表現される． 第1種の値として，箱に格納された実体値（entity value）がある．

  • Int: Java整数のコンテナ
  • Dbl: Java実数のコンテナ
  • Bool: true, false．Javaブール値のコンテナ．
  • Char：Unicode文字．Integerコンテナに格納．
  • Str: 二重引用符，単一引用符，スラッシュ引用符．f,q,r演算子．Java文字列のコンテナ．

第2種の値として，箱のポインタである参照 Ref（reference value）がある．

  • IntRef: Java整数のコンテナ
  • DblRef: Java実数のコンテナ
  • BoolRef: true, false．Javaブール値のコンテナ．
  • CharRef: Unicode文字．Integerコンテナに格納．
  • StrRef: 二重引用符，単一引用符，スラッシュ引用符．f,q,r演算子．Java文字列のコンテナ．
  • FrameRef: メモリ上に格納されたオブジェクトのフレームの開始番地を保持する．プロセスや，構造体の実体のポインタとして用いる．

そのアクセスと解釈のために，値は次の情報をもつ：

  • ポインタ： メモリ上で，その値を格納するコンテナの番地を表すJava整数．値は生存期間があり，aliveとdeadの二つの状態をもつ．
  • 値型： 型を表すenum型．実体型と参照型で異なる型をもつ．

将来的には次の構造値の導入を考えている．実装方法は，今後の課題である．

  • タプル： 多値返り値
  • 辞書： ディスパッチャー，構造体，モジュール
  • リスト： 値の列
