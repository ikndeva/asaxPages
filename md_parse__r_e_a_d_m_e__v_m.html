JVM

プリミティブ型

Javaのプリミティブ型は次の通り。XMachineでは、☆印のものを実装予定。

  • byte, 8bit ☆,
  • short, 16bit,
  • int, 32bit ☆,
  • long, 64bit,
  • char, 16bit ☆, as 符号なし2バイトint (UTF16)
  • float, 32bit,
  • double ☆, 64bit,
  • boolean ☆, 8bit, as byte, {true = 1, false = 0}.

XMでは、Javaのreference型の代わりに、次のプリミティブ型を追加する。

  • string, 32bit ☆, as a reference to a consecutive memory of char's.

オペランドスタック

オペランドスタックは基本型のFILOのスタックである。以下では、ボトムを左側に、トップを右側に、次のように表す。

O_Stack: op_1 op_2 ... op_N

機械語呼び出しで、スタックに引数op_1 op_2 ... op_Nを積む場合は、上の順で引数順にプッシュして得られる順序で積むと約束する。

JVMのオペランドスタックは次のように実装されている(*1)。

  • 各エントリ一つの領域サイズは、32bit int 型で表現
  • 例外として、long, double: 32bit int二つで表現
  • オペランドスタック上の値は、push後は変更できない。つまりオペランドスタック上の値はアトミックである。

*1) ソース: https://ja.wikipedia.org/wiki/オペランドスタック

定数プール

文字列や配列などのオブジェクトは、定数プールに積む。文字列の初期化

string s1 = "A", s2 = "ABC", t = "Apple", name = "Bill Atkinson";

は次のようにメモリ上で表現される。

string s1 = 0, s2 = 1, t = 2, name = 3
index string
0 "A"
1 "ABC"
2 "Apple"
3 "Bill Atkinson"

仮想機械語命令

仮想機械語命令 (instruction)について

  • プログラムコードは、バイトの列として表現される。
typedef vector<byte> Code

プログラムの各要素は次の二種類のいずれか:

  • オプコード Opcode: 仮想機械語命令を表す1バイトの値 0..255
  • オペランド Operand: 引数を表す1個以上のバイトの列

プログラムにおいて、機械語呼び出しは次の形の可変長部分列として表される。

opcode operand_0 ... operand_K

プログラムにおいて、各オプコードopcodeはそれに続く引数の数 $K\ge 0$ を決定するので、プログラム全体は、prefix code をなす。よって、先頭から一意にデコードできる。

オペランドの引数は、一つ以上のバイト列で表される. オペランドのバイト数は、opcodeが定める。

  • 8bit値 op0: そのまま 8bit 値を表す.
  • 16bit値 op0 op1: 次の16 bit 値を表す (op0 << 8) + op1.
  • 32bit値 op0 op1 op2 op3: 次の32 bit 値を表す (op0 << 24) + (op0 << 16) + (op0 << 8) + op1.
